# TypeScript Logger 系统设计文档

## 1. 项目简介
本项目是一个基于 TypeScript 设计的轻量级日志系统（Logger）。它旨在为 Web 应用提供统一、类型安全且可扩展的日志记录功能。系统核心采用了**策略模式 (Strategy Pattern)**，实现了日志记录逻辑与输出方式的解耦，既满足当前控制台输出的需求，又为未来写入文件或发送网络请求预留了完美的扩展接口。

## 2. 设计理念

本系统的架构设计遵循以下核心原则：

### 2.1 策略模式 (Strategy Pattern)
这是本系统的核心设计亮点。
- **问题**：传统的日志系统往往将“生成日志”和“输出日志”（如 `console.log` 或 `fs.writeFile`）硬编码在一起，导致难以切换输出目标。
- **解决方案**：定义了一个通用的 `LogOutputStrategy` 接口。`Logger` 类只负责生成日志条目（包含时间戳、级别、消息），而具体的“写入动作”委托给注入的策略对象。
- **优势**：符合**开闭原则 (Open/Closed Principle)**。如果需要增加新的输出方式（例如写入数据库），只需新增一个实现该接口的类，无需修改 `Logger` 的核心代码。

### 2.2 单一职责原则 (Single Responsibility Principle)
代码结构被严格拆分为多个文件，每个文件只负责一件事：
- `logger.types.ts`: 仅负责类型定义和接口契约。
- `logger.strategies.ts`: 仅负责具体的输出实现（控制台或文件模拟）。
- `logger.ts`: 仅负责日志级别过滤、数据序列化和流程控制。
- `index.ts`: 仅负责业务场景演示和测试。

### 2.3 类型安全与健壮性
- 利用 TypeScript 的强类型系统，严格限制日志级别只能是 `'VERBOSE' | 'INFO' | 'WARNING' | 'ERROR'`。
- 内置对象序列化保护机制，自动处理对象转字符串，并捕获序列化错误，防止因数据格式问题导致程序崩溃。

## 3. 快速开始与使用示例

### 3.1 基础用法 (默认控制台输出)
默认情况下，Logger 会将日志输出到浏览器控制台，且默认级别为 `INFO`（忽略 `VERBOSE`）。

```typescript
import { Logger } from './logger';

// 实例化 Logger (默认使用 ConsoleOutputStrategy)
const logger = new Logger();

// 输出 INFO 级别日志 (正常显示)
logger.info('应用程序已启动');

// 输出 WARNING 级别日志 (正常显示)
logger.warning('内存使用率超过 80%');

// 输出 VERBOSE 级别日志 (默认被忽略，因为当前级别是 INFO)
logger.verbose('调试详细信息...'); 

// 自动序列化对象
logger.error({ code: 500, message: '数据库连接失败' });
```

### 3.2 动态调整日志级别
可以在运行时动态调整日志记录的详细程度。

```typescript
import { Logger, LogLevel } from './logger';

const logger = new Logger();

// 初始为 INFO，VERBOSE 不输出
logger.verbose('这条不会显示');

// 调整为 VERBOSE，所有日志均输出
logger.setLevel(LogLevel.VERBOSE);
logger.verbose('✅ 现在这条调试信息也能看到了');

// 调整为 ERROR，仅输出错误
logger.setLevel(LogLevel.ERROR);
logger.info('❌ 这条普通信息被忽略');
logger.error('✅ 只有错误会显示');
```

### 3.3 扩展用法 (模拟文件输出)
通过注入 `FileOutputStrategy`，轻松切换输出目标至文件（当前为模拟实现）。

```typescript
import { Logger, LogLevel } from './logger';
import { FileOutputStrategy } from './logger.strategies';

// 1. 创建文件输出策略，指定文件路径
const fileStrategy = new FileOutputStrategy('./logs/system.log');

// 2. 实例化 Logger 并注入策略
const fileLogger = new Logger(LogLevel.INFO, fileStrategy);

// 3. 记录日志 (实际会调用模拟的 NativeFileWriteSync)
fileLogger.info('用户登录成功 - ID: 1001');
fileLogger.error('支付网关超时 - OrderID: 9527');

// 观察控制台：你会看到 [File IO Mock] 折叠项，显示模拟的文件写入内容
```

## 4. 架构扩展性说明

本系统如何支持未来的功能扩展？

### 4.1 扩展至真实文件写入
当前代码使用了 `NativeFileWriteSync` 进行模拟。在未来需要真实文件 I/O 时（例如在 Node.js 环境），只需修改 `FileOutputStrategy` 的实现：

```typescript
// 未来在 Node.js 环境中的真实实现示例
import * as fs from 'fs';

export class RealFileOutputStrategy implements LogOutputStrategy {
  write(entry: LogEntry): void {
    const logLine = JSON.stringify(entry) + '\n';
    // 真实的同步写入
    fs.appendFileSync(this.filePath, logLine); 
  }
}
```
**注意**：`Logger` 类本身不需要任何修改。

### 4.2 扩展至网络上报
如果需要将错误日志实时发送到监控服务器（如 Sentry 或自研后端），只需新建一个策略类：

```typescript
export class HttpOutputStrategy implements LogOutputStrategy {
  private apiUrl: string;
  
  constructor(url: string) { this.apiUrl = url; }

  write(entry: LogEntry): void {
    if (entry.level === 'ERROR') {
      fetch(this.apiUrl, {
        method: 'POST',
        body: JSON.stringify(entry)
      });
    }
  }
}
```
使用时只需：`new Logger(LogLevel.INFO, new HttpOutputStrategy('...'))`。

### 4.3 扩展新的日志级别
若需增加 `CRITICAL` 级别：
1. 在 `logger.types.ts` 中添加 `'CRITICAL'` 到类型联合体和 `LogLevel` 常量。
2. 在 `logger.ts` 的 `getLevelPriority` 函数中为其分配优先级数值。
3. 在 `ConsoleOutputStrategy` 中添加对应的 `console` 处理方法。

## 5. 项目结构

```text
LOGGER-TASK
├── src
│   ├── index.ts              # 入口文件：包含所有测试场景和使用示例
│   ├── logger.ts             # 核心类：Logger 实现，负责流程控制
│   ├── logger.strategies.ts  # 策略实现：ConsoleOutputStrategy, FileOutputStrategy
│   └── logger.types.ts       # 类型定义：接口、枚举、类型别名
├── package.json
└── tsconfig.json
```

## 6. 运行说明

确保已安装 Node.js 和 npm。

1. **安装依赖** (本项目无第三方依赖，但需初始化环境):
   ```bash
   npm install
   ```

2. **运行测试**:
   ```bash
   npx ts-node src/index.ts
   ```
   *或者如果配置了 npm script:*
   ```bash
   npm start
   ```

运行后，请在控制台观察不同场景下的日志输出及 `[File IO Mock]` 的模拟文件写入效果。
